# Contributing

## Code Quality

- Keep APIs small: This promotes modularity and makes code easier to maintain.
- Prefer composition over inheritance: This reduces coupling and increases flexibility in design.

## Testing

- Add tests: Ensure all new features and bug fixes are covered by comprehensive unit and integration tests.

## Documentation

- Update docs: Keep the documentation in sync with code changes to ensure clarity for users and contributors.

## Dependencies

- Keep dependencies minimal: Limit third-party libraries to reduce complexity and potential security risks.
